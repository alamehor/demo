spaCy.pl

Devloping tools for polish language processing in spaCy.

Our goal is to extend spaCy, a popular production-ready NLP library, to fully support polish language.
We are dedicated to using state-of-the art techniques for polish language processing, which are demonstrated on this website.

Development status

Feature Development status Details
Tokenization Version: Final
Status: Done
Release: Merged
 Current version is available on the master branch of spaCy, our contribution included an improved list of stop words for Polish.
For implementation details see our pull request
Lexical attributes Version: Final
Status: Needs bug fixes
Release: Merged
 Some minor bugs in stop words still require fixing, they will added to out next pull request.
Initial release of attributes was added to spaCy in the pull request with tokenizer
Lemmatization Version: POC
Status: Done
Release: In progress
 POC lemmatizer (based on ispell spell-checker rules) has just been implemented and will be added to out next pull request.
Working branch for our lemmatizer
Word vectors Version: POC
Status: Done
Release: None
 We currently use word2vec embeddings trained on NKJP corpus. This is only a proof of concept, because we expect Facebook's fasttext vectors to be significantly more accurate.
The vectors will be released as part of a model package and are currently developed on our utils repo
POS Tagger Version: POC
Status: Done
Release: None
 Initial POS Tagger with 89% tag accuracy. We still need to tune its hyperparameters and find the optimal way of merging some of the original NKJP tags for better balance.
POS Tagger will be released as part of a model package and is currently developed on our utils repo
Dependency Parser Version: POC
Status: Done
Release: None
 Reasonably good initial model with validation metrics reaching 91% UAS and 86% LAS. Similarily to POS Tagger, it still needs to be tuned before release.
Dependency parser will be released as part of a model package and is currently developed on our utils repo
NER Version: POC
Status: In progress
Release: None
 Work in progress, this is the last model we have to train before releasing the complete package.
NER will be released as part of a model package and is currently developed on our utils repo
