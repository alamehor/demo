Advanced text search demo

Achieving better search matches thanks to correct tokenization and lemmatization.

Lemmatization is the process of grouping together forms of a word so they can be analysed as a single item, identified by the word's lemma, or dictionary form.

In spaCy, every language has a built-in tokenizer, which allows us to filter meaningful words from the text. We can then use lemmatization on these tokens to process a search query - instead of looking at direct matches, we look for lemma matches. That way, we can not only find the word in a form it was typed in by the user, but in other forms as well.

To present how this process works, we have implemented a search tool that looks for words in an invocation to "Pan Tadeusz" - a classic Polish work of art written by Adam Mickiewicz. The search tool tries to match words provided in a form in two ways:

by comparing words directly and by comparing words' lemmas

Because Polish language is very complex, searching direct matches of words is often pointless. We display both kinds of matches so that you can judge how greatly can this word processing technique improve search results. Of course, this kind of functionality would rather be implemented on a database such as Elasticsearch. This demo is only intended to showcase how much of a difference lemmatization can make.
